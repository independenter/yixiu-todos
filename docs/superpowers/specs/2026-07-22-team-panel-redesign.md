# 团队面板重建设计

> 将团队面板从"员工管理"升级为"以项目为中心的团队协作视图"。
> 核心概念：多人支撑一事、一人支撑多事、全局/项目双优先级联动。

---

## 1. 设计目标

- **以项目为中心**：顶部分组展示各项目及其任务，取代当前以员工为中心的卡片布局
- **多人支撑一事**：一个项目多人协作，任务分配到人，集中展示
- **一人支撑多事**：底部展示每人跨项目的总投入，冲突时告警
- **双优先级联动**：全局优先级（所有任务统一排序）与项目内优先级（项目维度排序）自动同步
- **拖拽排序**：任务行可拖动调整优先级，全局和项目维度同步更新
- **时间选择**：顶部时间范围选择器（近3天/本周/本月/本季度/全部）

## 3. 数据模型扩展

### 3.1 新建 `projects` 表

```sql
CREATE TABLE IF NOT EXISTS projects (
    id          TEXT PRIMARY KEY,
    name        TEXT NOT NULL,
    description TEXT DEFAULT '',
    priority    INTEGER NOT NULL DEFAULT 2,  -- 1=紧急 2=高 3=中 4=低
    status      TEXT NOT NULL DEFAULT 'active', -- active/paused/done
    created_at  TEXT NOT NULL,
    updated_at  TEXT NOT NULL
);
```

### 3.2 `employee_tasks` 表扩充

```sql
ALTER TABLE employee_tasks ADD COLUMN project_id TEXT;
ALTER TABLE employee_tasks ADD COLUMN global_priority INTEGER NOT NULL DEFAULT 0;
ALTER TABLE employee_tasks ADD COLUMN project_priority INTEGER NOT NULL DEFAULT 0;
```

- `global_priority`：全局排位（所有项目统一排序，0=最高）
- `project_priority`：项目内排位（同一项目内排序，0=最高）
- 两者联动：修改全局优先级时，自动重算项目内优先级；反之亦然

### 3.3 冲突检测逻辑

个人在**同一时间片**内分配到的任务精力总和超过 100% 时：

| 条件 | 行为 |
|------|------|
| 超载、且该任务为项目内低优先级 | 标记为 **风险**，提示调整时间或加人 |
| 超载、但所有任务均为高优先级 | 标记为 **冲突**，要求加班或强行显示 |
| 未超载 | 正常显示 |

## 4. 功能模块

### 4.1 项目总览

- 顶部卡片网格，每个项目一个卡片
- 显示：项目名、优先级(P1-P4)、成员数、完成进度
- 点击选中 → 下方显示项目详情
- 颜色标记：🟢正常 ⚠️超载 🟡接近上限

### 4.2 项目详情 / 任务列表

- 选中项目后展开，显示该项目的所有任务
- 每行：任务名 + 负责人 + 精力占用 + 优先级星级 + 风险状态
- **可拖拽排序**：拖到新位置时同时更新 `global_priority` 和 `project_priority`
- "+ 分配任务" 按钮：弹窗选择员工、填写任务信息
- "📋 成员" 按钮：查看该项目所有参与成员的投入情况

### 4.3 全局 vs 项目内优先级联动

**更新规则：**

- 用户在**全局视图**拖动任务 → 更新所有任务的 `global_priority` → 自动重算各项目内的 `project_priority`
- 用户在**项目内视图**拖动任务 → 更新该项目内任务的 `project_priority` → 自动映射到 `global_priority`（保持相对顺序）

**排序算法：**

```
global_priority = 位置序号（0-indexed，整个团队）
project_priority = 在项目内的位置序号（0-indexed）
```

更新时，两个字段保持单调递增，通过重算受影响的范围来实现同步。

### 4.4 个人投入概览

- 底部折叠区域，展开后显示每个员工的跨项目投入
- 每行：姓名 → 各项目任务及精力 → 总和 → 状态标记
- 与个人看板的精力时间轴互不冲突，此处为"当前投入快照"

### 4.5 时间选择器

- 顶部按钮组：近3天 / 本周 / 本月 / 本季度 / 全部
- 影响：项目任务列表的起止时间过滤、个人投入的计算范围

## 7. 边界情况

- 无项目时：显示空状态提示 + "新建项目"按钮
- 项目无任务时：显示"暂无任务，点击分配"
- 员工不归属于任何项目时：在个人投入中显示"未分配"，但不阻塞
- 全局优先级更新时，需批量 UPDATE 避免排序冲突
- 删除项目时，下属任务可选择迁移或删除
